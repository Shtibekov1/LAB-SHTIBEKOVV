- Показ/скрытие пароля.
- Сбор данных через FormData.
- Закрытие модального окна по клику вне области диалога.

## Вопросы для защиты (с ответами)

### 1. Элемент `<dialog>`, методы showModal() и show()
`<dialog>` — нативное модальное окно.
- `show()` — немодальное.
- `showModal()` — модальное с блокировкой фона (backdrop).

### 2. Фазы события и опции обработчика
Фазы:
1. Capturing
2. Target
3. Bubbling

Опции:
- `capture`
- `once`
- `passive`

### 3. target и currentTarget
- `event.target` — реальный инициатор события.
- `event.currentTarget` — элемент, на котором висит обработчик.

### 4. FormData
Сбор данных из формы.
Методы: `get`, `set`, `append`, `delete`.
Перебор:
```js
for (const [key, val] of fd.entries()) {}
```

### 5. Предотвращение стандартного поведения
`event.preventDefault()` — отменяет отправку формы, переходы и т.п.

### 6. Ограничивающие атрибуты форм
`required`, `minlength`, `maxlength`, `min`, `max`, `pattern`, `type`.

### 7. Цель валидации клиента
Повысить удобство, предотвратить ошибки до отправки, снизить нагрузку.

### 8. :user-valid и :user-invalid
Отражают состояние **после взаимодействия пользователя**.  
Отличаются от `:valid` / `:invalid`, которые срабатывают всегда.

### 9. ARIA: aria-describedby, aria-live, aria-invalid
- `aria-describedby` — связывает поле с блоком ошибки.
- `aria-live="polite"` — мягкое обновление текста.
- `aria-invalid="true"` — поле содержит ошибку.

### 10. Constraint Validation API
`input.validity.*`, `checkValidity()`, `reportValidity()`,
`setCustomValidity()`, `validity.valid`.
